## Навигация (pageHistory)

```
home → config → data → checklist → photos → checklist → complete → home
                         ↑_____________________________|
```

- `header-back` всегда показывает предыдущую страницу
- `checklist` → клик по секции → `photos`
- `photos` → "Сохранить" → `checklist`
- `checklist` → "Завершить отчёт" → `complete`

## Управление отчётом

```
1. Создание:    config → data → checklist (currentReport.id = generateId)
2. Редактирование: клик по черновику → checklist
3. Фото:        клик по типу → камера / КЕ-модалка
4. Завершение:   finishReport() → showComplete() → buildZipBlob()
5. Экспорт:     sendReport() → navigator.share | downloadBlob()
6. Хранение:    IndexedDB — автосохранение каждые 2с
```

## Типы объектов (data.js)

| Тип | Название | Оборудование | Разделитель |
|-----|----------|-------------|-------------|
| mk  | Магнит Косметик | Касса, ТД, ТСД, УКС, МП | (пусто) |
| mm  | Магнит Супермаркет | Кассовая зона, КСО, ТД, ТСД, УКС, Весы, МП | ` # ` |
| ma  | Магнит Аптека | Кассовая зона, ТД, ТСД, УКС | ` # ` |

## Ключевые переменные

| Переменная | Тип | Назначение |
|-----------|-----|-----------|
| `currentReport` | object | Текущий отчёт (IndexedDB) |
| `currentSectionIndex` | number | Индекс текущей секции |
| `selectedPhotoType` | string | ID выбранного типа фото |
| `selectedType` | string | "mk" / "mm" / "ma" |
| `equipmentCounts` | object | { kassa: 2, td: 1, ... } |
| `deferredPrompt` | Event | beforeinstallprompt |
| `cachedZipBlob` | Blob | Предсобранный ZIP-архив |
| `videoStream` | MediaStream | Поток камеры |
| `scanTimer` | number | Таймер сканирования |
| `scanCooldown` | boolean | Защита от дубликатов |

## PhotoType флаги

| Флаг | Описание | Действие |
|------|---------|----------|
| `isKE` | Инвентарный номер | Открывает сканер штрихкодов |
| `isSN` | Серийный номер | Открывает сканер штрихкодов |
| `multi` | Можно несколько фото | До `maxPhotos` штук |
