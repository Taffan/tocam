# Фото Отчёты — История разработки

## Статус: Активная разработка

## Контекст

**Цель:** Создать PWA для техников, которые создают фото-отчёты о техническом обслуживании торговых точек сети "Магнит".

**Формат отчёта:** Фотографии + XLSX файл с перечнем (точно как в референсных папках `МК Заподборье`, `ММ Бителево`).

**Ключевое требование:** Работа полностью офлайн, т.к. техники работают в подвалах/складах без интернета.

---

## Что сделано

### Инфраструктура
- Репозиторий GitHub: https://github.com/Taffan/tocam
- GitHub Pages хостинг: https://taffan.github.io/tocam/
- PWA с Service Worker (кэширование всех файлов)
- Manifest для установки на главный экран
- Capacitor подготовлен (fallback вариант)

### Функциональность
- **3 типа отчётов:** МК (Косметик), ММ (Супермаркет), МА (Аптека)
- **Конфигурация оборудования:** выбор количества касс, принтеров, сканеров и т.д.
- **Чек-листы:** обязательные пункты проверки
- **Фотографирование:** камера + галерея, до 4 фото на тип
- **Сканер штрих-кодов:**
  - BarcodeDetector API (Android Chrome 83+) — без интернета
  - ZXing fallback
  - Звуковой сигнал + вибрация
  - Cooldown 2 сек
  - Валидация формата (EAN-13 = 13 цифр)
- **IndexedDB:** автосохранение черновиков
- **Экспорт:**
  - ZIP архив (JSZip)
  - XLSX отчёт (SheetJS)
  - Фото + КЕ.txt + JSON

### Дизайн
- Apple-style интерфейс
- Красный (#DC0000) + белый корпоративные цвета
- SVG иконки (без эмодзи)
- Адаптив для мобильных устройств

## На чём остановились

### Проблема: Web Share API на Android

**Симптомы:**
- На iOS: "Поделиться" работает — файл прикрепляется к сообщению
- На Android: Telegram не получает файл (только текст)

**Причина:** Android Web Share API ограничен — большинство приложений (Telegram, Email, VK) не принимают файлы через Web API. Это ограничение платформы Android, не браузера или PWA.

**Что попробовано:**
- `navigator.canShare()` → не всегда точное
- Deep links (`tg://`, `whatsapp://`) → не работают из PWA
- Web Share API с файлами → работает только на iOS и некоторых Android (Samsung)

**Статус:** Решено fallback-ом — при неудаче файл скачивается и пользователь отправляет вручную.

### Проблема: 404 после установки PWA

**Симптомы:** В браузере всё работает, но установленное приложение показывает 404.

**Причина:** Service Worker закэшировал 404 ответ для корневого пути.

**Решение:** Обновлён Service Worker (v3) с принудительным fetch через сеть и fallback на кэш.

**Статус:** ✅ Исправлено.

## Архитектура

```
PWA (GitHub Pages)
    │
    ├── index.html          ← интерфейс
    ├── styles.css          ← Apple-style стили
    ├── app.js              ← вся логика
    ├── data.js             ← TYPE_CONFIGS + PHOTO_TYPES
    ├── sw.js               ← Service Worker (кэш)
    └── manifest.json       ← PWA манифест

Локальное хранилище:
    ├── IndexedDB          ← черновики отчётов
    ├── Service Worker     ← кэш файлов + CDN библиотек
    └── LocalStorage       ← настройки

CDN зависимости:
    ├── JSZip               ← ZIP архивы
    ├── SheetJS (xlsx)     ← Excel экспорт
    └── ZXing              ← сканер штрих-кодов (fallback)
```
