# Фото Отчёты — История разработки

## Статус: Активная разработка — все основные функции реализованы

## Контекст

**Цель:** PWA для техников сети «Магнит» — создание фото-отчётов о техническом обслуживании.

**Формат:** Фотографии + XLSX + TXT с кодами.

---

## Что сделано

### Функциональность

#### Типы отчётов
- **МК** (Косметик)
- **ММ** (Супермаркет) — добавлен РМД
- **МА** (Аптека)

#### Оборудование
Касса, КСО, ТД, ТСД, УКС, Весы, МП, Прочее, **РМД** (Рабочее место директора)

#### Фото-типы для каждого оборудования
- **Фото** — обычные (multi: до 4-10)
- **КЕ** — инвентарные номера (сканер ШК)
- **СН** — серийные номера (сканер ШК)
- **Подсказки** — кнопка "?" раскрывает инструкцию для каждого типа фото

#### Интерфейс фотоотчёта
```
Страница фото
├── Список типов фото (Фото / КЕ / СН)
│   ├── Тип с фото: галочка ✓
│   ├── Тип без фото: круг
│   └── Кнопка "?" — раскрывает подсказку
├── Сетка thumbnails
│   ├── Клик → открывает галерею
│   └── ✕ удалить
└── Галерея (модалка)
    ├── Header с кнопкой "← Назад"
    ├── Кнопка "Сделать фото"
    ├── Thumbnails (клик → preview) ← НЕ РАБОТАЕТ
    ├── Кнопка "Готово"
    └── Крестик на thumbnail → удалить
```

#### Сканер
- BarcodeDetector API (Android Chrome 83+, iOS Safari 16.4+)
- ZXing fallback (старые iOS, старые Android)
- Интерактивный режим: жёлтые рамки на всех обнаруженных ШК → выбор → трекинг
- Подтверждённый код: зелёная рамка + "нажмите Сделать фото"
- Потеря кода: красная рамка + предупреждение
- **КЕ** — только 13-значные EAN-13 или 4-значные коды (цифры)
- **СН** — любые ШК и QR-коды
- `isKECode()` — общая функция валидации для KE
- `cachedOverlayCodes` — кэш набора кодов (без мерцания при каждом кадре)
- Звук + вибрация
- iOS: `!video.videoWidth` guard для гонки при старте камеры

#### Экспорт
```
ZIP/
├── XLSX
├── Коды.txt
├── КЕ/ (инвентарные номера)
├── Серийные номера/ (серийные номера)
└── N# *.jpg
```

### Валидация
- Фото — обязательны
- КЕ и СН — опциональны
- XLSX и TXT: "X из Y" для КЕ и СН

### Подсказки для фото
Все типы фото получили подсказки из документации:
- Кнопка "?" справа от названия типа фото
- При нажатии раскрывается подсказка с инструкцией
- Добавлены в data.js (hint property)
- Рендерятся в app.js (renderPhotoTypes)

## Архитектура

### IIFE структура

Весь код app.js обёрнут в IIFE. Переменные `currentReport`, `currentSectionIndex` недоступны из глобальной области. Это создаёт проблему при использовании onclick атрибутов. Решение: функции `__galleryTap`, `galleryPhotoTap` экспортируются в `window.*`.

### Настройки

- `getPhotoMaxSize()` — возвращает 1024/1920/3264px по `localStorage`
- `getScannerRes()` — возвращает объект разрешения для `getUserMedia()`
- `loadSettings()` — применяет настройки при загрузке страницы (качество, тёмная тема)
- `saveSetting(key, value)` — сохраняет в `localStorage`

### Взаимодействие

- **Короткий тап** на тип фото → камера/сканер/превью
- **Долгое нажатие (1.5 сек)** на тип фото → выбор фото из галереи устройства
- Реализация: `pointerdown` (мышь) / `touchstart` (тач) → `setTimeout(1500)` → `gallery-input.click()`; `longPressActivated` флаг блокирует click
- `clearLongPress()` на `pointerup`/`pointercancel`/`touchend`/`touchcancel` — глобальные слушатели, зарегистрированы один раз

### Поток данных

```
IndexedDB (PhotoReportsDB_v6)
    ↓ loadDrafts()
Главная (черновики)
    ↓ click draft → showChecklist()
Чек-лист (секции с прогрессом)
    ↓ click section → showPhotoSection()
Страница фото
    ├→ renderPhotoTypes() — список типов
    ├→ renderSectionPhotos() — thumbnails
    ├→ openPhotoGallery() — модалка с камерой
    ├→ galleryPhotoTap() — preview из галереи
    └→ openPhotoPreview() — preview напрямую
         ↓
    ZIP (buildZipBlob)
         ↓
    Экспорт (navigator.share / download)
```
