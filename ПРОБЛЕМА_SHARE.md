# Проблема: navigator.share({ files }) не открывает шеру на ПК

## Симптомы
- После нажатия «Отправить отчёт» на **ПК (Yandex Browser, Chrome на Windows)** — открывается диалог сохранения файла вместо системного окна «Поделиться» (share sheet)
- На **Android Chrome** — ошибка `Permission denied`
- На **iOS Safari** — не тестировалось, предположительно работает

## Ожидаемое поведение
Должно открываться системное окно «Поделиться» (share sheet) с выбором приложения: Telegram, WhatsApp, Email, и т.д., с прикреплённым ZIP-архивом.

## Фактическое поведение
На ПК браузер предлагает **скачать файл** (через download dialog), а не поделиться им через другое приложение.

## Почему так происходит

### 1. navigator.share не предназначен для десктопа
`navigator.share()` — это **Mobile Web API**. На десктопных браузерах (Windows, macOS, Linux) он:

- Поддерживает только `{ title, text, url }` (текст/ссылку)
- **НЕ поддерживает `{ files }`** на Windows/ macOS / Linux никогда
- При попытке `navigator.share({ files: [file] })` на десктопе браузер:
  - Либо выбрасывает `TypeError` / `NotAllowedError`
  - Либо (как в нашем случае) ведёт себя как download — потому что Chromium на Win32 не умеет открывать share sheet для файлов

### 2. На Android Permission Denied
На Android Chrome 89+ `navigator.share({ files })` **должен работать**, но на устройстве пользователя вылетает `Permission denied`. Возможные причины:

- **Старая версия Chrome** — нужен Chrome 89+ для поддержки файлов в share
- **Старая версия Android** — нужен Android 10+
- **Отозвано разрешение Storage / Notification** — Chrome на Android требует разрешения для шаринга файлов
- **PWA в standalone mode** — некоторые версии Android Chrome в PWA-режиме ограничивают share
- **VPN** — пользователь упоминал, что проблема может быть связана с VPN

### 3. Текущий fallback
В текущей реализации `sendReport()`:
1. Пытается `navigator.share({ files })`
2. Если ошибка (любая, кроме AbortError) — **скачивает файл** через `downloadBlob()` в папку Загрузки

Это «лечит» симптом, но не решает проблему — пользователь всё равно не может отправить файл напрямую из приложения.

## Возможные решения

### Вариант A: Использовать Web Share Target
- `Web Share Target` — это **получение** файлов (когда ваше PWA указано как цель для шеры из других приложений)
- Не решает проблему — нам нужно **отправлять**, а не принимать

### Вариант B: Собрать нативное приложение через Capacitor/Cordova
- **Capacitor** (от Ionic) — оборачивает PWA в нативное приложение
- Позволяет использовать нативные API шаринга:
  - Android: `Intent.ACTION_SEND` (ShareCompat)
  - iOS: `UIActivityViewController`
- Надёжно работает и на ПК, и на мобильных устройствах
- Минусы: нужно собирать APK/IPA, публиковать в магазины или распространять как файл

### Вариант C: Открыть файл через iframe / a[download] + инструкция
- Скачать файл (как сейчас)
- Показать инструкцию: «Откройте файл в проводнике и отправьте через Telegram / Email»
- Самый простой, но неудобный для пользователя

### Вариант D: navigator.share без файлов (только текст)
- Отправлять через `navigator.share({ title, text, url })` ссылку на сайт или имя файла
- Файл при этом должен быть загружен отдельно
- Не подходит — пользователь хочет отправить именно ZIP-архив как вложение

### Вариант E: Использовать сторонний сервис для загрузки файлов
- Загрузить архив на сервер (например, Firebase Storage, S3)
- Отправить ссылку через `navigator.share({ url })`
- Минусы: нужен бэкенд, сложнее поддержка офлайн

## Вывод
На десктопе `navigator.share({ files })` **никогда не будет работать**. На Android — работает нестабильно (зависит от версии Chrome и Android, разрешений, VPN).

**Рекомендация**: собрать нативное приложение через Capacitor для полноценной поддержки шаринга файлов на всех платформах.
