# 📁 Дерево отчёта — структура данных и ZIP-архива

## Структура currentReport (IndexedDB)

```
currentReport (объект в IndexedDB: PhotoReportsDB_v6.reports)
│
├── id:             string   ("l9x2m8r4k1...")
├── created:        string   (ISO дата)
├── modified:       string   (ISO дата)
├── status:         string   ("draft" | "completed")
├── completedAt:    string   (ISO дата, только если status="completed")
├── reportName:     string   ("Заподборье")
├── objectType:     string   ("mk" | "mm" | "ma")
├── equipmentCounts: object  ({ kassa: 2, td: 1, uks: 1, mp: 1, rmd: 1 })
├── date:           string   ("2026-07-14")
├── technician:     string   ("Иванов И.И.")
├── comment:        string   (опционально)
├── keCodes:        string[] (["1234567890123", "5901234567890"])
├── snCodes:        string[] (["QR-code-data", "CODE128-data"])
│
└── sections:       Section[]
```

## Структура секции

```
Section
│
├── id:          string   ("kassa_1")
├── name:        string   ("Касса 1")
├── prefix:      string   ("Касса")
├── num:         number   (1)
├── type:        string   ("kassa" | "td" | "tsd" | "uks" | "mp"
│                          | "kassa_zona" | "kso" | "vesi" | "prochee")
├── status:      string   ("pending")
│
├── photoTypes:  PhotoType[]
│   │
│   └── PhotoType
│       ├── id:        string   ("obshiy_vid")
│       ├── name:      string   ("Общий Вид")
│       ├── filename:  string   ("Касса 1 Общий Вид.jpg")
│       ├── multi:     boolean  (true — можно несколько фото)
│       ├── maxPhotos: number   (4 — лимит для multi)
│       ├── isKE:      boolean  (true — фото инвентарного номера)
│       └── isSN:      boolean  (true — фото серийного номера)
│
└── photos:      Photo[]

    └── Photo
        ├── id:          string   (generateId)
        ├── typeId:      string   ("obshiy_vid")
        ├── photoNumber: number   (1, 2, 3... для multi)
        ├── dataUrl:     string   (data:image/jpeg;base64,...)
        └── timestamp:   string   (ISO дата)
```

## Типы фото по оборудованию

### Касса (kassa)

```
Фото (обычные):
  ├── Общий Вид                       [multi: до 4]
  │   └── Видно расположение оборудования (ККМ, сканер ШК, клавиатура, монитор, ТБО, дисплей, весы).
  ├── Общий Вид (покупатель)          [multi: до 4]
  ├── Общий Вид Оборудование          [multi: до 4]
  │   └── Видно расположение оборудования на столе.
  ├── ККТ                             [multi: до 4]
  │   └── ККТ имеет все элементы. Снятый Х-отчет: дата, качество печати, без срезов, сработавший отрезчик.
  ├── Фото Чека                       [multi: до 4]
  │   └── Чек с читаемым текстом, дата, качество печати.
  ├── Маркировка БП                   [multi: до 4]
  │   └── Видно маркировку на блоках питания.
  └── Маркировка оборудования         [multi: до 4]
      └── Видно отчетливо маркировку оборудования.

КЕ | СН (инвентарные номера | серийные номера):
  ├── КЕ ККТ / СН ККТ
  │   └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
  ├── КЕ ИБП / СН ИБП
  │   └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
  ├── КЕ ПК / СН ПК
  │   └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
  ├── КЕ Пин Пад / СН Пин Пад
  │   └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
  ├── КЕ Сканер ШК / СН Сканер ШК
  └── КЕ Ручной Сканер ШК / СН Ручной Сканер ШК
      └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
```

### УКС (uks)

```
Фото:
  ├── Фото общего вида УКС (со стороны АММ)         [multi: до 4]
  │   └── Видно все оборудование, установленное на УКС, без каких-либо срезов.
  │       ПК и ИБП должны быть установлены на полке/подставке.
  ├── Фото общего вида УКС (вид сзади)               [multi: до 4]

  ├── Фото общего вида УКС (вид сзади нижняя часть)  [multi: до 4]
  │   └── Видно расположение ИБП, системных блоков. Недопустимо на полу.
  ├── Фото индикации на шлюзе                        [multi: до 4]
  │   └── Видно, что на голосовом шлюзе отключен МР! Фото не вырвано из контекста.
  ├── Фото обслуженного ПКД                          [multi: до 4]
  │   └── Видно отсутствие грязи и пыли внутри системного блока.
  ├── Фото обслуженного Серверного ПК                [multi: до 4]
  │   └── Видно отсутствие грязи и пыли внутри системного блока.
  ├── Фото сетевого оборудования                     [multi: до 4]
  │   └── Видно маркировку проводов и хаба для оборудования (кроме ТД).
  ├── Фото сетевого оборудования для ТД              [multi: до 4]
  │   └── Видно маркировку проводов и хаба для ТД.
  ├── Фото сетевого оборудования доступа в интернет  [multi: до 4]
  │   └── Видно маркировку роутеров (основной и резерв) и проводов.
  ├── Фото переключателя                             [multi: до 4]
  │   └── Видно маркировку на переключателе.
  ├── Фото маркировки БП в блоках розеток           [multi: до 4]
  │   └── Видно маркировку на всех блоках питания.
  └── Фото ключа ЕГАИС                              [multi: до 4]
      └── Видно бирку с названием магазина, ключ пристегнут к корпусу ПКД.

КЕ | СН:
  ├── Фото КЕ Серверного ПК / СН Серверного ПК
  │   └── Текст маркировок читаемым; Изображение четким; Наклейка + оборудование видно.
  ├── Фото КЕ ИБП Сервер / СН ИБП Сервер
  ├── Фото КЕ ПКД / СН ПКД
  ├── Фото КЕ ИБП ПКД / СН ИБП ПКД
  ├── Фото КЕ Монитор / СН Монитор
  ├── Фото КЕ Голосовой шлюз / СН Голосовой шлюз

```

## Валидация

- **Обычные фото** — обязательны для сохранения секции
- **КЕ и СН** — опциональны (можно сохранить отчёт без них)
- В статистику XLSX и TXT выводится "X из Y" для КЕ и СН

## Формат имён файлов в ZIP

| Условие | Формат | Пример |
|---------|--------|--------|
| Все типы (МК/ММ/МА/ГМ) | `{prefix} {num} # {filename} {N}.jpg` | `Кассовая зона 1 # ККТ 1.jpg` |
| Без номера (Прочее) | `Прочее # {filename} {N}.jpg` | `Прочее # Планограмма 1.jpg` |
| КЕ-фото | `{filename}` → папка `КЕ/` | `КЕ/Кассовая зона 1 # КЕ ККТ 1.jpg` |
| СН-фото | `{filename}` → папка `Серийные номера/` | `Серийные номера/Кассовая зона 1 # СН ККТ 1.jpg` |
| Обычное фото | `{filename} {N}.jpg` → корень | `Кассовая зона 1 # ККТ 1.jpg` |
| Счётчик | Всегда добавляется `{N}` (начиная с 1) | ` 1`, ` 2`, ` 3`... |
